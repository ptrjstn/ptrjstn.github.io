# ptrjstn Website

Dieses Repo bündelt jetzt alles unter einem Dach:

- `/` → gemeinsame Landingpage
- `/Playground/` → Playground
- `/Portfolio/` → Portfolio
- `/Storybooks/` → Storybooks

## Lokal

Einfach den Ordner `Website/` mit einem Static Server öffnen oder direkt `index.html` im Browser laden.

## GitHub Pages

Für das Repo `ptrjstn.github.io` kannst du GitHub Pages direkt aus dem `main`-Branch und dem Root-Verzeichnis ausliefern.

Die Datei `.nojekyll` sorgt dafür, dass GitHub Pages die Ordnerstruktur unverändert ausliefert.
